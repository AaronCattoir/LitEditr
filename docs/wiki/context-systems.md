# Context management systems (map)

[← Wiki home](./README.md)

This project uses several **separate** context stacks. They differ by **consumer** (pipeline vs. chat vs. UI), **source of truth** (live graph state vs. SQLite vs. filesystem), and **whether retrieval is RAG** (none of these paths use vector RAG today—story chat is explicit manuscript slices only).

---

## Orchestration (`run_analysis`)

Aligned with [`AGENT_CONTEXT.md`](../../AGENT_CONTEXT.md):

**Full run (`run_analysis`)** in [`graph.py`](../../src/narrative_dag/graph.py) — **imperative** orchestration: LLM chunker → plot overview → character map → calibration on the first chunk → **for each chunk**, a Python loop over the pipeline node functions (after `defense`, `editor_judge` and `evidence_synthesizer` run in parallel, then `elasticity`). Static Mermaid for documentation is generated by [`scripts/generate_graph_visuals.py`](../../scripts/generate_graph_visuals.py) (`artifacts/graphs/`).

**Chat / interaction** — [`nodes/interaction.py`](../../src/narrative_dag/nodes/interaction.py) exposes node callables (`run_judge_explain`, `run_judge_reconsider`); the service wires LLM calls directly (no graph library).

## 1. Per-chunk editorial pipeline (`run_analysis`)

| Piece | Role | Where |
|-------|------|--------|
| **Chunks** | Partition of the manuscript (`Chunk` list with global char offsets). | [`nodes/ingestion.py`](../../src/narrative_dag/nodes/ingestion.py) — `run_chunker`, `chunk_document`, LLM narrative chunking, chapter-first path |
| **`ContextWindow`** | Target chunk + sliding **previous/next** neighbor chunks + optional `global_summary`. | `build_context_window` / `run_context_builder` in `ingestion.py` |
| **`PromptContext`** | Normalized bundle for prompts: neighbors, plot/stakes/themes, emotional/narrative curves, character DB, last few **chunk judgments**. | [`prompt_context.py`](../../src/narrative_dag/prompt_context.py) — `build_prompt_context`, `format_prompt_context` |
| **`DocumentState`** | Rolling **global editor memory** (voice baseline merge, emotional curve, narrative map, character voice map, plot overview, etc.) updated each chunk. | [`nodes/representation.py`](../../src/narrative_dag/nodes/representation.py) — `run_document_state_builder` / `document_state_builder` |
| **Pipeline state** | Plain `dict[str, Any]` merged across nodes (see [`AGENT_CONTEXT.md`](../../AGENT_CONTEXT.md) state contract). | [`graph.py`](../../src/narrative_dag/graph.py) orchestration |

**Flow (conceptual):** chunks → `context_window` → representation nodes → detectors / critic / defense → judge / evidence / elasticity.

**Example:** [Pipeline: `ContextWindow` and prompt text](./examples/01-pipeline-context-window.md)

## 2. Persisted run artifacts and interaction bundles

| Piece | Role | Where |
|-------|------|--------|
| **`RunStore`** | Saves run metadata, per-chunk JSON artifacts, latest `document_state`. | [`store/run_store.py`](../../src/narrative_dag/store/run_store.py) |
| **`ContextBundle`** | **Per run + chunk**: `target_chunk`, `context_window`, `document_state`, detector/critic/defense/judgment results—**not** from chat history. | **`serialize_story_wide_for_api`** | Subset of `DocumentState` (plot, characters, maps, voice) as JSON for APIs and chat system prompts. | `run_store.py` |

## 3. Story chat (Inkblot) — explicit manuscript context (no RAG)

| Piece | Role | Where |
|-------|------|--------|
| **`build_explicit_context`** | Chooses **chunk list** or **chapter segment** (or manuscript prefix if no chapters), applies **word budget**, returns **manifest + excerpt text**. | [`explicit_context.py`](../../src/narrative_dag/explicit_context.py) |
| **Pet soul** | Static markdown seed (global + optional per-`document_id` override); sections parsed for system prompt. | [`pet_soul.py`](../../src/narrative_dag/pet_soul.py); [`docs/pet/`](../pet/PET_SOUL.md), optional `docs/pet/stories/<document_id>.md`, env `EDITR_PET_SOUL_DIR` |
| **Persona snapshot** | **Deterministic** JSON + optional **LLM snapshot** (alignment, visual model) + style policy; versioned in DB. | [`persona/engine.py`](../../src/narrative_dag/persona/engine.py), [`store/persona_store.py`](../../src/narrative_dag/store/persona_store.py), [`persona/refresh_job.py`](../../src/narrative_dag/persona/refresh_job.py) |
| **`run_inkblot_chat`** | Composes system prompt (soul + deterministic persona + voice + optional story-wide + manifest) + **rolling turn history** + user message; manuscript excerpt in the **user** message body. | [`story_chat.py`](../../src/narrative_dag/story_chat.py) |
| **Chunk-scoped judgment context** | When `chunk_ids` are set (explicit section scope), loads **latest run for that revision** and attaches compact **editor judgment + critic + advocate** from `RunStore` chunk artifacts (`build_inkblot_judgment_context`). Advisory framing: context, not a hard rule. Chapter/prefix scopes do not auto-inject this. | [`service.py`](../../src/narrative_dag/service.py) (`story_chat`), [`story_chat.py`](../../src/narrative_dag/story_chat.py) |
| **`story_wide_from_document_state` / `serialize_story_wide_for_api`** | Injects analyzed global story fields into chat when available. | `story_chat.py`, `run_store.py` |
| **`compact_older_turns_for_summary`** | Deterministic truncation when turn window overflows (config `STORY_CHAT_ACTIVE_TURNS`). | `story_chat.py`, [`config.py`](../../src/narrative_dag/config.py) |
| **`StoryChatStore`** | SQLite: sessions + turns (server-side chat memory). | [`store/story_chat_store.py`](../../src/narrative_dag/store/story_chat_store.py) |
| **Frontend `storyChatStorage`** | Browser **session id** per document (localStorage), not full transcript; optional **`pinned_chunk_ids`** after quick-coach → Inkblot so the panel scopes the next sends to that section. | [`Editr-FrontEnd/src/lib/storyChatStorage.ts`](../../Editr-FrontEnd/src/lib/storyChatStorage.ts) |

**Config:** `STORY_CHAT_MAX_WORDS_DEFAULT`, `STORY_CHAT_ACTIVE_TURNS` in `config.py`.

**Example:** [Inkblot: persona snapshots and message assembly](./examples/03-inkblot-persona-and-messages.md)

## Quick comparison

| System | Primary question it answers | Storage |
|--------|----------------------------|---------|
| Pipeline `ContextWindow` | What text surrounds **this chunk** for detectors/judges? | In-memory pipeline `dict` during `run_analysis` |
| `PromptContext` | What structured editorial facts go into **this stage’s** prompt string? | Derived from state each call |
| `ContextBundle` | What did we **persist** for explain/reconsider for **this run + chunk**? | SQLite via `RunStore` |
| Explicit story-chat context | What manuscript **excerpt** grounds Inkblot **this turn**? | Built from `DocumentStore` revision text + scope params |
| Persona + soul | How should Inkblot **behave** and sound (stable + refreshed)? | Filesystem soul + SQLite persona snapshots/events |
| Story-chat turns | What was **said** before in this chat? | SQLite + short client session id |
